⚠ Your master list of reps — replaces the old DV Helper tab. Positions persist across uploads. A green dot means the rep has payments this period; a red flag means their position is unassigned and pay can't be calculated.

How It Works

The roster is your source of truth — it replaces the old "DV Helper" tab. You set each rep's position once, and it sticks forever.

Auto-import: When you upload a CSV, new reps are added to the roster automatically. Any rep with payments but no position gets flagged red so you know to fix them before the pay calc runs.

Positions: Directors get the raw Revenue amount. Everyone else uses the Pay Rates table based on their role.

Rep Name Aliases

⚠ Any rep whose CSV name matches a "From" value gets remapped to the "To" value everywhere in the app — the Office view, breakdowns, exports, and Positions list. Matching is case & accent insensitive.

How It Works

Use aliases when a rep's name changes in real life but the CRM hasn't caught up yet — gender transitions, marriage name changes, typos, nickname standardization, etc.

Example: "Grace Spreen" → "Thomas Crenshaw" will relabel every transaction where the CSV says "Grace Spreen" as "Thomas Crenshaw" — totals, positions, exports, everything.

Matching ignores case, accents, and extra whitespace. So Grace Spreen, grace spreen, and Gráce  Spreen all match the same rule.
